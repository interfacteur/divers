Variables : déclaration, affectation, exécution

EN "use strict"

une déclaration de variable vaut "synchroniquement" sur sa portée, par effet du "hoisting" elle existe dès le début de l'exécution :
	que sa déclaration soit instruite avant ou après (la déclaration n'étant pas encore exécutée) de l'exécution,
	que sa déclaration soit même exécutable ou non exécutable : au-delà d'un return, dans une condition non remplie etc.

une affectation vaut "diachroniquement" sur sa portée, c'est-à-dire qu'elle n'a de valeur que lorsque l'affectation est exécutée
	par "hoisting" la variable existe comme === undefined jusqu'à l'instant de l'exécution de l'affectation


AVANT "use strict"

affecter une valeur à une variable non déclarée crée une variable globale : existant après l'affectation, devant apparaître "diachroniquement" dans le contexte global
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/var :
"Declared variables are created before any code is executed. Undeclared variables do not exist until the code assigning to them is executed."

PS : delete fonctionne avec cet emploi périmé des variables (alors propriétés de window)
